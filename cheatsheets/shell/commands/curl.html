$ man curl
NAME
       curl - transfer a URL

SYNOPSIS
       curl [options / URLs]

DESCRIPTION
       curl  is a tool to transfer data from or to a server, using one of the supported protocols (DICT, FILE, FTP, FTPS, GOPHER, HTTP, HTTPS,
       IMAP, IMAPS, LDAP, LDAPS, POP3, POP3S, RTMP, RTSP, SCP, SFTP, SMB, SMBS, SMTP, SMTPS, TELNET and TFTP). The command is designed to work
       without user interaction.

       curl  offers  a busload of useful tricks like proxy support, user authentication, FTP upload, HTTP post, SSL connections, cookies, file
       transfer resume, Metalink, and more. As you will see below, the number of features will make your head spin!

       curl is powered by libcurl for all transfer-related features. See libcurl(3) for details.

CLI

Usage


curl [OPTIONS...] URL

Flags

flag

description

example

-I, --head

Show headers only

‚ÄúShow document info only‚Äù

-v

Verbose

-L

Follow redirects.

-H, --header HEADER

Request headers.

-H 'Content-Type: application/json'

-X, --request TYPE

Request type. Note that passing data with -d automatically generates a POST.

-X POST

-d, --data DATA

Pass data in a request. Typically JSON or form data in a POST.

--data 'foo=bar;bazz=123

--data '{"foo": "bar", "bazz": 123}' -H 'Content-Type: application/json'

-F FIELD=VALUE

Send field

-F 'foo=bar' 'bazz=123'

-u, --user USER:PASSWORD

Server user and password

-A, --user-agent NAME

Send User-Agent value

curl -A "Mozilla/5.0 (X11; Linux x86_64; rv:60.0) Gecko/20100101 Firefox/81.0" https://example.com/"

-O

Output file with resource‚Äôs name. No argument required. The name will be inferred from the URL - useful for when downloading a zip file for example.

curl -O https://example.com/abc/def.zip

Now available as def.zip.

-o OUTPUT_PATH

Save resource with desired name

curl URL -o foo.txt

Files

Provide a path to a file.

Basic upload

curl -F 'upload=@my_file.txt' URL

Image

curl -F 'image=@my_image.png' URL

Headers

Protocol response headers.

Using -I flag, only the headers will be shown. No content.

This can be useful for debugging an API or a server-rendered page, where you don‚Äôt get about the content (which can be long).
