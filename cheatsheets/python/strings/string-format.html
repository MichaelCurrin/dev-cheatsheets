String format

Cheatsheets / Python / Strings / String format

Guide to using str.format and f-string syntax.

Basics

Implied order

"{} {}".format(foo, bar)

Explicit order

"{0} {1}".format(foo, bar)

Named parameters

"{foo} {bar}".format(foo=foo, bar=bar)

F-string

The rules of the str.format approach work with the newer F-string syntax, introduced in Python 3.6. This replaces the need for the older named parameters approach above.

f"{foo} {bar}"

Number formatting done with both styles:

x = 1234

"{:10,}".format(x)
# '     1,234'

f"{x:10,}"
# '     1,234'

Modifiers

The default behavior is to call .__format__() internally. Here we use repr and str.

If you omit the zero below, it might work or might give an error IndexError: Replacement index 1 out of range for positional args tuple.

repr

Using __repr__(). This is similar but different to using str.

'{0!r}'

Use ASCII modifier, since Python 3.

'{0!a}'

Example

class Data:

    def __str__(self):
        return 'str'

    def __repr__(self):
        return 'repr'
        
        
'{0!s} {0!r}'.format(Data())
# 'str repr'

e.g.

class Data:

    def __repr__(self):
        return 'r√§pr'
        
        
# Interactive console.
'{0!r} {0!a}'.format(Data())
'r√§pr r\\xe4pr'

# Printing.
print('{0!r} {0!a}'.format(Data()))
# r√§pr r\xe4pr

With an integer, the value will appear the same.

i = 2021

f"{i} {i!r} {i!a}"
# "2021 '2021' '2021'"

But with a string, you‚Äôll get quotes for r and a.

s = '2021'

f"{s} {s!r} {s!a}"
# "2021 '2021' '2021'"

Alignment

With no direction set, the default for strings it to align left. Number alignment is covered in Numbers.

'{:CHARS}'

Align left.

'{:<CHARS}'

Align right with padding.

'{:>CHARS}'

Align center.

'{:^CHARS}'

e.g.

"{:<4}".format("ab")
'ab  '

"{:>4}".format("ab")
'  ab'

"{:^4}".format("ab")
' ab '
# Odd number for even characters mean extra space on the right.
"{:^5}".format("ab")
' ab  '

Numbers

Align

With no spacing set, Python can‚Äôt know how to align right, so there will be no padding.

If you set d as the type and specify total character places, you‚Äôll get aligned right by default.

"{:CHARSd}"

e.g. Setting a total of 4 spaces, which adds two spaces before two-digit number.

"{:4d}".format(42)
# '  42'

"{42:4d}"
# '  42'

See also Add padding below.

Separators

Make a long number readable using a separator like a comma.

"{:SEPARATOR}"

e.g. Use "{:,}" or `‚Äù‚Äù.

"{:,}".format(12345678910)
# 12,345,678,910

Round a float

Load Pi as our float.

import math


math.pi
# 3.141592653589793

Default - implied as "{:7.6f}". i.e. 7 digits with 6 after the decimal point.

"{:f}".format(math.pi)
# '3.141593'

Or using f-strings.

f"{math.pi:f}"
'3.141593'

Custom format - 3 digits, 2 after the decimal point.

"{:3.2f}".format(math.pi)
# '3.14'

Add padding

For numbers, the Alignment here defaults to right with > implied.

Integer

Leading zeroes padding.

"{:04d}".format(4)
# '0004'

'{:04d}'.format(42)
# '0042'

Out of interest, align left.

'{:<04d}'.format(42)
'4200'

Use white space padding and align to right.

'{:4d}'.format(42)
# '  42'

Float

Leading zeroes.

"{:06.2f}".format(math.pi)
# '003.14'
